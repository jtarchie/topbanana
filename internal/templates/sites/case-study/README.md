# Case study

## Purpose
Long-form customer story — kicker, headline, three stat cards, then Problem / Solution / Results / pull quote. Picked when the user wants to tell one customer's story (marketing-led B2B sites, agency portfolios).

## What ships
- `skeleton/index.html` — editorial layout with placeholder stat cards and section headings.

## Checks
- `index.html` must contain `<h1` — every case study needs a headline that names the customer.
